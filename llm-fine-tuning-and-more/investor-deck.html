sunshine acoustic model (SAM)

the ears — the world's first high-fidelity ear for indian pediatric speech

problem: the "autocorrect" trap

standard ASR (whisper/google) prioritizes grammar over raw sound

child says "wabbit" → generic AI outputs "rabbit"

erases the clinical symptom we're trying to treat

solution: specialist LoRA adapters

fine-tuned on open-weight models for phonetic fidelity · LoRA paper

adapterdetects
articulationmicro-substitutions (/w/ vs /r/)
fluencyrhythm breaks, blocks (b-b-ball)
toddlerhigh-pitch, regional code-switching
indian englisht/d retroflex, indian phonetics

research: Kid-Whisper (AAAI'24) · Child ASR (ICASSP'24)

example

child says: "i see a wabbit"

SAM hears: /aɪ/ /siː/ /ə/ /w/ /æ/ /b/ /ɪ/ /t/

(generic AI would output "rabbit" — useless)

evals

metrictarget
PER (phoneme error rate)<10%
error preservation>95%
latency<500ms

moat

proprietary library of disordered child speech in indian accents — can't be scraped from the internet

sunshine classification model (SCM)

the judge — automated clinical diagnostics

problem: precision labeling

SAM provides the raw sound: /w/

the session defines the clinical target: /r/

how do we know if the child got it wrong?

solution: algorithmic alignment

deterministic math (computational linguistics), not neural networks · logic doesn't hallucinate

techniquepurpose
levenshtein distancephoneme-to-phoneme comparison
indian phonetic mapweighted similarity matrix for indian sounds
error classificationsubstitution, omission, addition, distortion

same input → same output. auditable, safe, explainable.

example

target: "rabbit" → /r/ /æ/ /b/ /ɪ/ /t/

heard: "wabbit" → /w/ /æ/ /b/ /ɪ/ /t/

SCM output: substitution /r/→/w/, severity 0.3

(not age-appropriate at 6+ → flag for intervention)

evals

metrictarget
diagnostic accuracy>95%
reliability (same input → same output)99%

moat

proprietary indian lexicon (the answer key) — US/UK dictionaries flag indian accents as errors (false positives), our phonetic map understands indian english

sunshine policy engine (SPE)

the clinical graph — the "operating system" for personalized care

problem: therapy is non-linear

therapy isn't a straight line; it's a complex web of dependencies

(e.g., jaw stability is a prerequisite for /r/)

static handbooks can't adapt to a child's changing mood or unique failure patterns in real-time

solution: navigational ai

componentfunction
the mapproprietary neo4j knowledge graph mapping full speech hierarchy: isolation → syllables → words → sentences
dynamic pathfindingqueries graph to generate menu of valid next steps (not rigid sequence)
personalization overlaycombines standard playbook (safety) + child history (interests) to filter menu

e.g., prioritizing "lateral move to 'space' theme" over "repetitive drills"

example

context: child failed 'rabbit' 3 times. interest: animals.

graph query: "find valid neighbours for /r/ substitution"

output menu:

scaffold_down_to_sound lateral_move_to_'tiger' break

evals

metrictarget
clinical safety score100%

zero unsafe recommendations generated

moat

first mover: first digitized clinical brain — transforming static textbooks + unstructured therapist wisdom into machine-readable logic

safety guardrails: graph physically blocks invalid paths — ai literally cannot suggest unsafe interventions

sunshine reasoning model (SRM)

the brain — the clinical strategist & empathetic coach

problem: the execution "gap"

the graph provides valid clinical options (scaffold, retry, break)

but selecting the right option and delivering it with the right tone requires synthesizing:

  • emotional state
  • longitudinal history
  • personal interests

rules cannot do this — only generative ai can

solution: behavioral cloning

llama 3 (8b) fine-tuned on session logs from our top 1% "star therapists"

stepfunction
inputreceives "menu of options" from graph + emotional context from SAM
reasoningchain-of-thought to simulate therapist decision process
outputchosen option + tone + delivery wrapped in child's interests

example

context: fail count is 3 (high risk). history shows he loves animals.

reasoning: "repeated failure requires a reset. i'll pick 'lateral move' but wrap it in a 'tiger' metaphor to match his interest."

"let's stop working! let's be tigers instead! can you growl? grrrr!"

metrictarget
engagement retentionsession completion ≥ human therapist

moat

digitized intuition: we haven't just taught ai to speak — we've trained it to make decisions like a senior clinician

persona lock: strictly trained for "parent coaching" — zero "internet noise" or hallucinations
